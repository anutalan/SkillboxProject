## 7. Критические бизнес сценарии 
### *0.1. Мобильное приложение. Установка и регистрация пользователя*

Actor: новый пользователь

Conditions:

- пользователь скачал мобильное приложение бренда

Steps:

1) Устанавливает приложение
2) Регистрируется в приложении с указанием страны пользователя и соглашаясь на ряд разрешений для приложения, включая согласие на сбор персональных данных и данных для мобильного приложения
3) Входит в приложение с указанием страны пользователя
4) Заполняет обязательные характеристики профиля с указанием спортивных предпочтений и характеристик тела
5) Видит доступные разделы приложения с социальной сетью, новостной лентой, тренировками, уведомлениями

### *0.2. Web приложение. Регистрация пользователя*

Actor: новый пользователь

Conditions:

- пользователь открыл браузер

Steps:

1) Указал адрес приложения и нажал опцию регистрации в приложении
2) Регистрируется в приложении с указанием страны пользователя и соглашаясь на ряд разрешений для приложения, включая согласие на сбор персональных данных
3) Входит в приложение 
4) Заполняет обязательные характеристики профиля с указанием спортивных предпочтений и характеристик тела
5) Видит доступные разделы приложения с социальной сетью, новостной лентой, тренировками, уведомлениями

### *1. Проведение тренировок*

Actor: постоянный пользователь

Conditions:

- пользователь авторизовался в приложении

Steps:

1) Начинает тренировку определенного вида спорта

2) *Для мобильных приложений только:* Данные о тренировке логируются и дополняются стандартными данными из приложений Андроида и IoS при их наличии

3) *Для мобильных приложений только:* Может подключить устройства, которые несинхронизированы с Андроид и IoS приложениями, и всю тренировку логировать и дополнять её данными из этих подключенных устройств, при их наличии 

4) Возможно смотреть онлайн трансляцию тренировки во время ее проведения
5) Пользователь сам завершает тренировку через какое-то время, завершая тренировки в связанных Андроид\ IoS\ устройствах с датчиками при их наличии

### *2. Просмотр сравнительных результатам по своим тренировкам*

Actor: постоянный пользователь

Conditions:

- завершил ряд тренировок

1) Может просматривать результаты отдельных своих тренировок, наглядо видеть сравнение своих показателей в течение времени

2) Для своих занятий данным видом спорта видит подборку товаров, инвентаря, которые:

- можно приобрести у бренда; 

- позволят проходить дальше тренировки с дополнительным оборудованием;

- помогут достигать лучших результатов;

- позволят не травмироваться;

- помогут находиться в более комфортных условиях на тренировках


### *3. Маркетинговые подсказки по выбору продукции*

Actor: постоянный пользователь

Conditions:

- в профиле заполнены данные о спортивной одежде, обуви, инвентаре

- на платформе есть данные о продолжительных тренировках

1) Получает уведомление о необходимости обновить:

- спортивную одежду  - если прошло фиксированное кол-во месяцевили кол-во пройденных занятий по определенным занятиям, сменился сезон

- обувь  - зафиксирован километраж на занятиях бегом, сменился сезон 

- инвентарь  - если прошло фиксированное кол-во месяцев по определенным занятиям, накатано определенное кол-во км (на велосипеде, лыжах..)

2) Для своих занятий данным видом спорта видит подборку товаров, инвентаря данного бренда.

### *4. Сетевая геймификация*

Actor: постоянный пользователь

Conditions:

- состоит в группах
- есть друзья в приложении

1) Получает приглашения от друзей, групп на участие в соревнованиях

2) Может ознакомиться с соревнованиями, куда его пригласили, в новостях, чатах, группах

3) Может внести в календарь данное соревнование

4) Может поучаствовать в соревновании, отмечая старт и завершение активности или получив от организаторов отметку об участии с полученными результатами

5) За участие в активности пользователь вознагражден очками, наградами

### *5. Масштабируемость сервиса при больших спортивных мероприятиях*

Actor: тысячи пользователей

Conditions:

- пользователи одновременно участвуют в спортивном мероприятии, который добавили в календарь

1) Тысячи пользователей одновмеренно отметили начало активности в одной локации или находясь онлайн в разных странах

2) Спортивное мероприятие обновляет новостной контент и результаты всех участников онлайн

3) Возможно онлайн вещание

4) Во время проведения мероприятия связь с платформой не теряется, нет задержек в ответах от платформы

5) Завершение мероприятия проходит штатно, кто-то самостоятельно заврешает активность, кто-то принудительно при завершении организаторами

6) Результаты доступны всем участникам соревнования

### *6. Создание новостной ленты*

Actor: пользователь-редактор 

Conditions:

- пользователь Web приложения

1) Открывает редактор новостей в браузере с множеством удобных функций редактирования текста и графического изображения, публикации рекламы бренда с ссылкой на магазин бренда или любое другое его приложение

2) Может указать ряд настроек видимости: временного графика отображения в новостной ленте, настройка пользователей, которым может быть интересна новость, меток-тегов, спортивным результатам пользователей

3) Может направить новость на редактирование или только просмотр другим пользователям, потом опубликовать

4) Все созданные новости может найти в общем разделе с удобной фильтрацией по меткам, администрировать их тоже через данный раздел


### *7. Администрирование новостной ленты с полным доступом*

Actor: пользователь-администратор

Conditions:

- пользователь Web приложения администраторов

1) Открывает раздел со списком всех новостей от разных редакторов
2) Может сделать неактивной любую новость, закрыть возможность комментировать ее другим пользователям
3) Может просматривать любую новость на предмет некорректного отображения контента, проигрывания медиа и аудио файлов
4) Может построить отчет о кол-ве новостного контента за определенный период
 

### *8. Администрирование в чате поддержки*

Actor: пользователь-администратор

Conditions:

- пользователь Web приложения администраторов

1) Открывает раздел со списком обращений пользователей
2) Может отвечать на обращения, помогая им решить их проблему, выставляя статус оказания услуги

### *9. Администрирование пользователей и социальной сети*

Actor: пользователь-администратор

Conditions:

- пользователь Web приложения администраторов

1) Открывает раздел со списком пользователей
2) Может изменять статус пользователей, временно блокировать, полностью удалять или восстанавливать
3) Может просматривать без ограничения видимости все группы платформы, изменять их статус, настройки
