## 9. НФТ

1) Доступность: Высокая доступность. Интерфейс должен быть рабочим круглосуточно, в любой день, позволяя тренироваться пользователям в любое время суток, общаться 24*7, создавать онлайн заказы круглосуточно при переходе в магазины бренда и смотреть статусы заказов в любое время. Время простоя не более 3 часов (время для обновлений). Желательно реализовать на время обновлений доступность приложения на просмотр данных, без их изменений.

2) Сопровождаемость: Время исправления крит ошибки максимум 3 дня.

3) Быстродействие: Обновление БД ядра с записью тренировки, показателей во время тренировки должно быть быстрым (согласно метрике производительности)  -  система должна предоставлять актуальные данные по get запросам, погрешность устаревания данных минимальна (последующий запрос через 1 секунду должен вернуть более актуальные данные).

4) Производительность: Страница отображается макс за 2 секунды для 95 процентов запросов. Нагрузка (соревнования, мероприятия на 100 000 пользователей) может увеличивать скорость обновления интерфейса, но не существенно, максимум на 20 процентов. 

5) Расширяемость: Зафиксирован прогнозируемый прирост пользователей в неделю (+2000 пользователей в первый год), месяц (+10 000 пользователей в первый год), год - пиковые нагрузки. Спрогнозирован прирост пользователей (после рекламных акций, праздников, сезонность): *10 раз. 

6) Безопасность: Пользовательские данные, включая персональные и медицинские показатели, должны быть защищены и зашифрованы, частично замаркированы в логах. 

7) Удобство использования: Интерфейс должен быть понятным, удобным ,чтобы пользователи хотели возвращаться в него, возможно несколько раз в день, общаться в соц сетях + конверсия от покупок должна показывать ,что пользователь часто туда заглядывает, доходит до создания заказов в магазинах бренда.

8) Надежность: Система позволяет хранить информацию за определенный период (10 лет). Данные по тренировкам должны сохраняться непрерывно для пользователя (задержки должны быть минимальными).

9) Тестируемость: Автотесты - покрытие 80 процентов сценариев пользователя. Количество ошибок меньше 90 процентов.

10) Аудит: Аудит действий пользователя, логирование, использование в ML для маркетингова анализа и качественной оценки UI, UX, для формирования предложений на покупку товаров, которые наиблее подходят пользователю, тем более если он им уже ранее интересовался. Логирование должно развиваться, чтобы выставлять SLA для команды поддержки.

11) Адаптивность: Поддержка разных платформ Windows, Linux, адаптивность к разным устройствам (телефонах, планшетах android, iOS, с разным расширением, с разными браузерами Chrome, Yandex, Firefox). Возможность установки приложения в разных облаках, в нескольких одновременно.
