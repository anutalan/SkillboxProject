## 6. План разработки
### 1. Этапы разработки 
#### *1. Определение целей разработки*
- Определение потребностей бизнеса, бизнес целей
- Формирование гипотез
- Определение целей разработки
- Определение стейкхолдеров
#### *2. Сбор бизнес требований*
- Сбор требований со стейкхолдеров/ пользователей/ бизнеса/ технических специалистов
- Анализ существующих конкурентов и их решений
- Формирование функциональных и нефункциональных требований
- Приоритизация требований
- Фиксация и формирование списка ограничений
- Фиксация и формирование списка рисков по проекту
- Фиксация атрибутов качества проекта
- Оценка стоимости разработки, выделение ресурсов
- Формирование команд
  - Поскольку в наличии есть большой штат сотрудников, то можно их сгруппировать по скилам, стеку и локации, а далее распределить уже сами сервисы между этими командами
- Декомпозиция требований на релизы (смотреть раздел 2)
#### *3. Системный и архитектурный анализ*
- Определение концептуальной архитектурной схемы
- Определение компонентов системы и их критичность, создание схем взаимодействия этих компонент
- Выбор технологичкеского стека
- Создание архитектуры с учетом аварийного восстановления данных, резеврвного копирования данных, переключения на резервный ЦОД
- Проектирование сценариев использования
- Создание прототипов UI, дизайна пользовательского интерфейса
- Проектирование схемы баз данных, основных сущностей, их связей
- Проектирование API компонентов, событий брокеров сообщений
- Ведение вертикальной документации по проекту
- Постоянный пресмотр бизнесовой декомпозиции требований на релизы согласно архитектуре и готовности системного анализа, оценке разработки
#### *4. Разработка*
- Использование метолодологий Agile, Scrum, Kanban
- Использование спринтов, проведение код ревью кода другими разработчиками
- Написание автотестов
- Ведение бэклога для техдолга, выделять 20 процентов на данные задачи и рефакторинг кода
#### *5. Тестирование*
- Юнит тесты для проверки модулей системы
- Тестирование на тестовом, интеграционном стендах
- Полное покрытие сценариями системы, обязательный регресс для каждого релиза
- Отдельный стенд для нагрузочного тестирования
- Создание тестовых сценариев и подготовка тестовых данных
- Тестирование требований и выставление аналитикам замечаний для актуализации документации
- Проведение приемочного тестирования заказчиком (UAT)
#### *6. Подготовительные работы перед релизом*
- Подготовка документации к дате релиза:
  - инструкций по установке,
  - пользовательской документации,
  - регистрации и публикации юридических договоров для запуска системы,
  - получение нужных лицензий и сертификатов для продуктового стендам,
  - заключение всех договоров о поддержке и использованию облачной инфраструктуры,
  - составление регламентов работы каждого отдела при возникновении инцидента на проде
- Подтоговка серверов, облачной инфраструктуры в срок, настройка CA/CD пайплайнов
- Организация поддержки пользователей, обучение поддержки функционалу системы
- Реализация возможной настройки канареечного релиза на узкий круг пользователей
#### *7. Поддержка*
- Фиксация проблем в проде и быстрое решение этих проблем согласно регламенту
- Мониторинг и логирование всех компонент во время работы и реагирование при возникновении ошибок, отказов
- Сбор метрик и их анализ, постоянное создание новых метрик/ их улучшение
- При наличии канареечного релиза можно и нужно собирать предложения от пользователей об улучшении бизнес процессов, сценариев приложожения

### 2. Декомпозиция требований

#### *1. MVP*

Срок: 9 - 12 месяцев

Цель: Проверить, что данное приложение с минимальным функционалом будет ценно и интересно для пользователей. 

Ограничения: в рамках одного облака, страны.

Реализация требований:

1) Пользователи: регистрация и создание профиля пользователя внутри одной страны, нагрузка до 100 000 пользователей.

2) Тренировки: сохранение тренировок пользователя, сравнительный анализ своих тренировок

3) Интеграция: минимальное подключение базовых функций Google Fit, Apple HealthKit, визуализация показателей устройств в приложении.

4) Геймификация: баллы, награды за достижения.

5) Социальная сеть: возможность добавлять друзей, общаться с другими пользователями.

6) Реклама бренда: лента новостей со статьями бренда, рекламой товаров.

7) Коммерция: интеграция с приоритетными магазинами по переходу в них по клику на рекламу бренда.

##### Критически важные компоненты и их важные сценарии:

- Сервис Пользователей
  - Должен позволять входить в систему и регистрировать новых пользователей
- Сервис Тренировок
  - Должен сохранять текущие тренировки без прерываний
- Сервис Новостей Рекламы
  - Должен позволять получать рекламу бренда с возможностью перехода в магазины бренда
- Сервис Социальной сети
  - Должен позволять общаться пользователям, чтобы они не уходили в другие соц сети, мессанджеры
- Сервис уведомлений
  - Должен позволять получать коды регистрации или подтверждения входа в систему

### *2. Развитие*

Срок: 12 - 18 месяцев

Цель: Расширение функционала с использованием ML, сравнительных стимуляций пользователя, развитем социальной сети. 

Ограничения: в рамках одного облака, нескольких стран.

Реализация требований:

1) Пользователи: нагрузка до 1 000 000 пользователей; пользователи из одной облачной инфраструктуры и из несколько стран.

2) Тренировки: создание плана тренировок, формирование календаря, его просмотр, редактирование.

3) Интеграция: получение новых показателей о тренировке из Google Fit, Apple HealthKit, визуализация показателей устройств в приложении.

4) Геймификация: награды за рейтинг среди других пользователей, визуализация рейтинга.

5) Социальная сеть: возможность вступать в группы, общаться с группой пользователей; возможность создавать соревнования-мероприятия.

6) Реклама бренда: акции на основе ML рекомендаций.

7) Коммерция: расширение кол-ва интеграций с магазинами бренда.


### *3. Итоговая версия*

Срок: 18 - 24 месяцев

Цель: Проверить, что данное приложение с развивающимся функционалом будет ценно и интересно для пользователей.

Реализация требований:

1) Пользователи: нагрузка до 100 000 000 пользователей; пользователи из нескольких облачных инфраструктур и стран.

2) Тренировки: поддержка большего кол-ва видов спорта, создание комбинированных программ по нескольким спортам одного пользователя с использованием ML сервиса.

3) Интеграция: с устройствами-датчиками, визуализация показателей устройств в приложении.

4) Социальная сеть: добавление привычных для социльных сетей доп. функциональности.

5) Реклама бренда: улучшение моделей ML.

6) Коммерция: использование BI отчетности.
